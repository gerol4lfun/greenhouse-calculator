# 🔄 Инструкция: Обновление Telegram бота на сервере

## ❌ Проблема

Бот на сервере (Railway) использует старую версию кода и не распознает города с ограничениями:
- Находит только 27 городов вместо 32
- Не распознает формат: `"Москва с 12.02, кроме 13.02, 14.02"`

## ✅ Решение

Нужно обновить код бота на сервере. Railway автоматически задеплоит новую версию после пуша в GitHub.

---

## 🚀 Пошаговая инструкция

### Шаг 1: Закоммитить изменения

Выполните в терминале (из корня проекта):

```bash
cd /Users/pavelkulcinskij/Downloads/kek
```

```bash
git add telegram-bot/
```

```bash
git add js/scripts.js
```

```bash
git add db/migrations/20260203_add_restrictions_to_delivery_dates.sql
```

```bash
git commit -m "Обновление парсера: поддержка ограничений доставки (формат с запятой)"
```

### Шаг 2: Отправить изменения в GitHub

```bash
git push origin main
```

### Шаг 3: Проверить деплой на Railway

1. Откройте https://railway.app
2. Войдите в аккаунт
3. Выберите проект с ботом
4. Перейдите во вкладку **"Deployments"** (слева)
5. Должен появиться новый деплой с вашим коммитом
6. Дождитесь завершения деплоя (статус "Active")

### Шаг 4: Перезапустить бота (если нужно)

Если деплой не запустился автоматически:

1. В Railway нажмите на сервис с ботом
2. Перейдите во вкладку **"Settings"**
3. Нажмите кнопку **"Redeploy"** (Переразвернуть)
4. Дождитесь завершения

### Шаг 5: Проверить работу бота

1. Откройте Telegram
2. Найдите вашего бота
3. Отправьте тестовое сообщение:

```
Москва с 12.02, кроме 13.02, 14.02
Питер с 9.02
```

4. Бот должен найти **2 записи** (не 1!)
5. В ответе должно быть:
   - `Москва - 12.02 (кроме 13.02, 14.02)`
   - `Санкт-Петербург - 9.02`

## 🔍 Проверка логов

Если что-то не работает, проверьте логи:

1. В Railway перейдите в **"Deployments"**
2. Выберите последний деплой
3. Нажмите **"View Logs"**
4. Проверьте, нет ли ошибок

## ⚠️ Важно

- Railway автоматически задеплоит новую версию после `git push`
- Если деплой не запустился, проверьте настройки GitHub в Railway
- Убедитесь, что в Railway подключен правильный репозиторий и ветка `main`

---

## 📞 Если не работает

1. **Проверьте, что код закоммичен:**
   ```bash
   git log --oneline -5
   ```
   Должен быть виден ваш коммит

2. **Проверьте, что код отправлен в GitHub:**
   - Откройте GitHub репозиторий
   - Проверьте, что файл `telegram-bot/parser.js` обновлен
   - Проверьте последний коммит

3. **Проверьте логи Railway:**
   - Могут быть ошибки при запуске
   - Проверьте переменные окружения

4. **Перезапустите бота вручную:**
   - В Railway: Settings → Redeploy
