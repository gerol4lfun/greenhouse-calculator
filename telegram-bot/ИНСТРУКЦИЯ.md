## ЧАСТЬ 1: Создание репозиториев на GitHub

### ШАГ 1: Создать репозиторий для КАЛЬКУЛЯТОРА

1. Откройте: https://github.com
2. Нажмите **"+"** (правый верхний угол) → **"New repository"**
3. Заполните:
   - **Repository name:** `teplitsa-calculator-main`
   - **Description:** (можно оставить пустым)
   - **Visibility:** выберите **Private** или **Public**

4. Нажмите **"Create repository"**

**✅ Готово! Репозиторий создан**

### ШАГ 7: Настроить запуск (МОЖНО ПРОПУСТИТЬ)

**✅ Railway автоматически использует команду из `package.json`!**

В вашем `package.json` уже есть скрипт `"start": "node index.js"`, поэтому Railway автоматически запустит бот командой `npm start`. 

**Этот шаг можно полностью пропустить!** Просто переходите к следующему шагу.

**✅ Готово! (пропущено - Railway всё делает автоматически)**

### ШАГ 8: Проверить деплой

1. Перейдите во вкладку **"Deployments"** (слева)
2. Найдите последний деплой (самый верхний)
3. Статус должен быть **"SUCCESS"** (зеленая галочка) или **"BUILDING"** (желтый индикатор)

**Если статус "SUCCESS":**
- ✅ Бот успешно задеплоен и работает!
- Переходите к проверке работы (ШАГ 9)

**Если статус "FAILED":**
- Откройте **"View Logs"** (или нажмите на деплой)
- Скопируйте текст ошибки и отправьте мне

**Если статус "BUILDING":**
- Подождите 1-2 минуты
- Обновите страницу
- Статус должен измениться на "SUCCESS"

**✅ Готово! Бот задеплоен и работает**

## 🎯 Как использовать бота:

**Каждый вечер отправляйте боту список дат в формате:**

```
Москва с 9.02
Тула с 9.02
Питер с 8.02
Воронеж с 12.02 (кроме 16)
Калуга с 9.02
Рязань с 9.02
```

**Можно отправлять много городов сразу (до 50+).**

**Форматы, которые понимает бот:**
- `Город с ДД.ММ` (например: `Москва с 9.02`)
- `Город с ДД.ММ (кроме ДД)` (например: `Воронеж с 12.02 (кроме 16)`)
- `Город с ДД.ММ (кроме ДД, ДД, ДД)` (например: `Тула с 9.02 (кроме 16, 20, 25)`)

**Бот автоматически:**
1. ✅ Распознает все города и даты
2. ✅ Обновит данные в Supabase
3. ✅ Отправит подробный отчет

**🚀 Готово к использованию!**
