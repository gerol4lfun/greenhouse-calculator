# ⚡ Быстрая проверка деплоя бота в Railway

## 🎯 Если сервис уже онлайн

Если вы видите, что сервис "Online" (зеленая точка), но нужно проверить, что новая версия загружена:

---

## 📍 Где найти деплойменты в Railway

### Вариант 1: Через карточку сервиса

1. **Найдите карточку с вашим ботом** (например, "teplitsa-calculator-main" или название вашего бота)
2. **Кликните на карточку** (не на "Online", а на саму карточку)
3. Откроется страница сервиса
4. **Сверху будут вкладки:** "Metrics", "Deployments", "Logs", "Settings"
5. **Кликните на "Deployments"** (Деплойменты)
6. Вы увидите список всех деплоев

### Вариант 2: Через меню проекта

1. В левом верхнем углу выберите проект (например, "calm-exploration")
2. В списке проектов выберите нужный
3. Найдите сервис с ботом
4. Кликните на него
5. Перейдите в "Deployments"

---

## ✅ Что проверить в Deployments

1. **Последний деплой должен быть сверху**
2. **Статус должен быть "SUCCESS"** (зеленая галочка) или "ACTIVE"
3. **Время деплоя** должно быть недавним (после вашего `git push`)
4. **Commit message** должен содержать ваш коммит

---

## 🔄 Если деплой старый (не обновился автоматически)

### Способ 1: Перезапустить вручную

1. В Railway откройте сервис с ботом
2. Перейдите во вкладку **"Settings"** (Настройки)
3. Прокрутите вниз до раздела **"Deploy"**
4. Нажмите кнопку **"Redeploy"** (Переразвернуть)
5. Дождитесь завершения (1-2 минуты)

### Способ 2: Проверить настройки GitHub

1. В Railway откройте сервис с ботом
2. Перейдите в **"Settings"**
3. Проверьте раздел **"Source"**
4. Убедитесь, что:
   - Подключен правильный репозиторий
   - Ветка = `main` (или `master`)
   - Автодеплой включен

---

## 🧪 Как проверить, что новая версия работает

### Тест 1: Проверить логи

1. В Railway откройте сервис с ботом
2. Перейдите в **"Logs"** (Логи)
3. Должны быть свежие записи (после перезапуска)
4. Не должно быть ошибок

### Тест 2: Проверить работу бота

Отправьте боту в Telegram:

```
Москва с 12.02, кроме 13.02, 14.02
Питер с 9.02
```

**Ожидаемый результат:**
- ✅ Найдено записей: **2** (не 1!)
- ✅ Москва - 12.02 (кроме 13.02, 14.02)
- ✅ Санкт-Петербург - 9.02

**Если бот находит только 1 запись** - старая версия еще работает, нужно перезапустить.

---

## 🚨 Если ничего не помогло

1. **Проверьте, что код действительно в GitHub:**
   - Откройте GitHub репозиторий
   - Перейдите в `telegram-bot/parser.js`
   - Проверьте, что в строке 34 есть: `const patternWithComma = /^(.+?)\s+с\s+(\d{1,2}\.\d{1,2})\s*,\s*кроме\s+(.+)$/i;`
   - Если этого нет - код не обновился в GitHub

2. **Проверьте переменные окружения:**
   - В Railway: Settings → Variables
   - Должны быть: `TELEGRAM_BOT_TOKEN`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`

3. **Проверьте логи на ошибки:**
   - В Railway: Logs
   - Ищите красные сообщения об ошибках
