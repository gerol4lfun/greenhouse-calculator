# 🧪 Алгоритм тестирования системы ограничений доставки

## ✅ Шаг 1: Проверка базы данных

### 1.1. Проверить наличие колонки `restrictions`

Откройте Supabase → SQL Editor и выполните:

```sql
SELECT column_name, data_type, is_nullable 
FROM information_schema.columns 
WHERE table_name = 'delivery_dates' AND column_name = 'restrictions';
```

**Ожидаемый результат:** Должна вернуться одна строка с информацией о колонке.

**Если колонки нет:**
```sql
ALTER TABLE delivery_dates 
ADD COLUMN IF NOT EXISTS restrictions TEXT DEFAULT NULL;

COMMENT ON COLUMN delivery_dates.restrictions IS 'Дни, когда доставка НЕ осуществляется. Формат: "13.02, 14.02" (даты в формате ДД.ММ)';
```

## ✅ Шаг 4: Проверка данных в Supabase

### 4.1. Проверить таблицу `delivery_dates`

Откройте Supabase → Table Editor → `delivery_dates`

**Проверьте следующие города:**

1. **Москва:**
   - `delivery_date` = `12.02`
   - `restrictions` = `13.02, 14.02`

2. **Тула:**
   - `delivery_date` = `12.02`
   - `restrictions` = `13.02, 14.02`

3. **Санкт-Петербург (или Питер):**
   - `delivery_date` = `09.02` или `9.02`
   - `restrictions` = `null` (пусто)

**Если данные не совпадают:**
- Проверьте логи бота в Railway
- Убедитесь, что бот успешно обновил данные (должно быть сообщение "✅ Обновление завершено!")

## ✅ Шаг 5: Проверка в калькуляторе

### 5.1. Проверить модальное окно "Актуальные даты доставки"

1. Откройте калькулятор в браузере
2. Нажмите кнопку **"Актуальные даты доставки"**
3. Найдите в таблице город **"Москва"**

**Ожидаемый результат:**
- **Город:** Москва
- **Дата доставки:** с 12.02.2026
- **Ограничения:** кроме 13.02, 14.02

**Если ограничения не отображаются:**
- Обновите страницу (F5)
- Проверьте консоль браузера (F12) на наличие ошибок

### 5.2. Проверить короткое КП

1. В калькуляторе выберите город **"Москва"**
2. Рассчитайте стоимость
3. Откройте вкладку **"Короткое КП"**

**Ожидаемый результат:**
В конце КП должна быть строка:
```
Ближайшая дата доставки — 12.02.2026, кроме 13.02, 14.02.
```

**Если ограничения не отображаются:**
- Проверьте, что в БД для Москвы есть `restrictions = "13.02, 14.02"`
- Обновите страницу и попробуйте снова

```

```
(города в одной строке через запятую)

```
Москва с 12.02, кроме 13/02, 14/02
```
(использование слэша вместо точки)

### ✅ Правильно:

```
Москва с 12.02, кроме 13.02, 14.02
Тула с 12.02, кроме 13.02, 14.02
```
(каждый город с новой строки)

## 🔍 Отладка (если что-то не работает)

### Проверить логи бота в Railway:

1. Railway → ваш проект → сервис с ботом
2. Перейдите в **"Logs"** (HTTP Logs)
3. Отправьте боту тестовое сообщение
4. Скопируйте логи и проверьте:
   - `🔍 Парсинг: обработано X строк` - должно быть правильное количество
   - `✅ Найдено с ограничениями` - должно быть для городов с "кроме"
   - `💾 Обновление: ...` - должно быть для каждого города
   - `📝 Данные для обновления` - должно содержать `restrictions`

### Проверить консоль браузера:

1. Откройте калькулятор
2. Нажмите F12 (открыть DevTools)
3. Перейдите во вкладку **"Console"**
4. Откройте модальное окно "Актуальные даты доставки"
5. Проверьте, нет ли ошибок (красные сообщения)
