# 🤖 Telegram бот для обновления дат доставки

**📍 Проект на Railway:** Telegram Bot - Delivery Dates

Бот автоматически обновляет даты доставки в Supabase на основе текста, который вы отправляете.

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка переменных окружения

Скопируйте `.env.example` в `.env` и заполните:

```bash
cp .env.example .env
```

Отредактируйте `.env`:
```
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
SUPABASE_URL=https://dyoibmfdohpvjltfaygr.supabase.co
SUPABASE_SERVICE_ROLE_KEY=ваш_service_role_key
```

### 3. Запуск

**Локально:**
```bash
npm start
```

**На продакшене:**
Бот автоматически запустится при деплое на хостинг.

---

## 📝 Использование

### Формат данных

Отправьте боту текст в таком формате:

```
Москва с 9.02
Тула с 9.02
Питер с 8.02
Воронеж с 12.02
```

С исключениями:
```
Москва с 9.02 (кроме 16)
Тула с 9.02 (кроме 16, 20)
```

### Что делает бот:

1. ✅ Парсит ваш текст
2. ✅ Показывает что будет обновлено
3. ✅ Обновляет данные в Supabase
4. ✅ Отправляет подтверждение

## ⚙️ Переменные окружения

| Переменная | Описание | Обязательно |
|------------|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather | ✅ Да |
| `SUPABASE_URL` | URL вашего Supabase проекта | ✅ Да |
| `SUPABASE_SERVICE_ROLE_KEY` | Service Role Key от Supabase | ✅ Да |
| `ADMIN_USER_ID` | ID пользователя Telegram (для ограничения доступа) | ❌ Нет |

---

## 📋 Поддерживаемые форматы

- `Город с ДД.ММ` - основная дата
- `Город с ДД.ММ (кроме ДД)` - с одним исключением
- `Город с ДД.ММ (кроме ДД, ДД, ДД)` - с несколькими исключениями

### Примеры:

```
Москва с 9.02
Санкт-Петербург с 8.02
Воронеж с 12.02 (кроме 16)
Тула с 9.02 (кроме 16, 20, 25)
```

## 🐛 Решение проблем

### Бот не отвечает
- Проверьте, что токен правильный
- Убедитесь, что бот запущен
- Проверьте логи на хостинге

### Ошибки при обновлении
- Проверьте Service Role Key
- Убедитесь, что таблица `delivery_dates` существует
- Проверьте права доступа в Supabase
