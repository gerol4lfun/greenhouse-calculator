## 🎯 ЦЕЛЬ ПРОЕКТА

Калькулятор стоимости теплиц с генерацией коммерческих предложений (КП) для менеджеров. Калькулятор позволяет:
- Рассчитывать стоимость теплицы с учетом всех параметров
- Генерировать короткое и длинное коммерческое предложение
- Добавлять грядки, дополнительные товары, сборку
- Рассчитывать доставку через Yandex Maps API
- Управлять подарками в зависимости от суммы заказа

## 📚 СТРУКТУРА ПРОЕКТА

### Основные файлы

| Файл | Назначение | Ключевые функции/компоненты |
|------|-----------|----------------------------|
| **index.html** | Главная страница | Авторизация, форма калькулятора, вкладки КП, админ-панель, модальные окна (FAQ, доставка, грядки, подарки) |
| **js/scripts.js** | Вся логика (~6900 строк) | `calculateGreenhouseCost()`, `generateShortOffer()`, `generateCommercialOffer()`, `updateGiftsBlock()`, `updateCommercialOffersWithGifts()`, авторизация через Supabase |
| **css/styles.css** | Стили интерфейса | Стили для калькулятора, вкладок КП, модальных окон, кнопок, форм, toast-уведомлений |

### Данные

| Источник | Что хранит | Где используется |
|----------|-----------|------------------|
| **Supabase (prices)** | Цены на теплицы по городам, формам, размерам | `loadCities()`, `onCityChange()`, `performCalculation()` |
| **Supabase (users)** | Логины и пароли менеджеров | `authenticate()`, админ-панель |
| **Supabase (delivery_dates)** | Даты доставки по городам | `loadDeliveryDate()`, модальное окно дат |
| **Supabase (knowledge_base)** | База знаний (25 карточек) | ❌ НЕ используется в калькуляторе |
| **Yandex Maps API** | Геокодирование адресов, расчёт маршрутов | `calculateDelivery()`, `calculateDeliveryDebounced()` |
| **Локально в коде** | Справочники: формы, цены сборки, дополнительные услуги, грядки, подарки | `availableForms`, `assemblyPrices`, `additionalServicesData`, `BEDS_DATA`, `GIFTS_DATA` |

## 🐛 ИЗВЕСТНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Исправленные проблемы

1. ✅ **Дублирование подарков в КП**
   - **Проблема:** При обновлении подарков в КП появлялись дубликаты блоков
   - **Решение:** Агрессивный цикл `while` с проверкой длины строки для удаления всех старых блоков

2. ✅ **Не учитывается количество одинаковых подарков**
   - **Проблема:** Если выбран 2 одинаковых подарка, показывался только один
   - **Решение:** Использование `Map` для подсчета количества, показывается "(2 шт)"

3. ✅ **Подарки не обновляются динамически в КП**
   - **Проблема:** Нужно было нажимать "Рассчитать" для обновления КП
   - **Решение:** Создана функция `updateCommercialOffersWithGifts()`, которая обновляет только секцию подарков

4. ✅ **Старые подарки остаются после изменения параметров**
   - **Проблема:** При изменении размера теплицы старые подарки не очищались
   - **Решение:** Очистка `localStorage.selectedGifts` в `updateGiftsBlockPreview()`

5. ✅ **"Автоматическая форточка" не скрывает второй слот сразу**
   - **Проблема:** Второй слот скрывался только после перерисовки
   - **Решение:** Прямое управление `style.display` в `onGiftChange()`

6. ✅ **Кнопка "Сбросить все фильтры" ломает выбор города**
   - **Проблема:** После сброса список городов становился пустым
   - **Решение:** Восстановление списка городов из кеша `citiesCache`

7. ✅ **Bug #1: Грядки исчезают из КП**
   - **Проблема:** В `performCalculation` перезаписывался `additionalProductsText`
   - **Решение:** Изменено на добавление: `additionalProductsText = (additionalProductsText ? additionalProductsText + '\n\n' : '') + productsText;`

8. ✅ **Bug #2: Неправильный расчет доставки во втором варианте**
   - **Проблема:** Доставка не передавалась в `computeFinalTotalPriceForVariant`
   - **Решение:** Добавлен параметр `deliveryPrice` в функцию

9. ✅ **Problem #5: Несогласованное округление**
   - **Проблема:** Округление применялось до добавления доставки
   - **Решение:** Округление применяется после добавления доставки
